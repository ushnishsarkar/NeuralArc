Step 1
Load a Dataset
Click the Dataset tab. Pick a built-in dataset (e.g. Iris) or drag-drop a CSV. The panel will profile it automatically — check the EDA tabs to understand your data.
Step 2
Bind to Graph
Click Bind to Graph → at the bottom of the Dataset panel. This pushes the feature count and class count into your Input and Output nodes automatically.
Step 3
Build Your Network
Go to the Build tab. Drag blocks from the left palette onto the canvas. Connect them by dragging from the right port → of one block to the ← left port of the next.
Step 4
Click ▶ Train
Press the green ▶ Train button. A live modal shows epoch-by-epoch loss and accuracy. When done, you are automatically taken to the Results tab.
🟢

Full Workflow
End-to-end pipeline from raw data to trained model
Phase 1 — Data
Open the Dataset tab and select a modality (Tabular / Image / Sequence) that matches your data. Upload a file or pick a built-in. The platform will auto-detect column types, compute statistics, and show class balance.
Use the Preprocessing controls on the left to clean the data: handle missing values, normalise features, encode categoricals, remove outliers, and balance classes. All changes are live — the EDA tabs update immediately.
Use the Validation EDA tab to run 7 automated checks on your data before training. Fix any red/amber issues first.
When satisfied, click Bind to Graph → to push the correct shapes into your network.
Phase 2 — Architecture
In the Build tab, select the matching modality (📊 Tab / 🖼 Img / 〰 Seq) using the strip at the top of the palette. Incompatible blocks are greyed out automatically.
Drag blocks onto the canvas. Connect them port-to-port. The Inspector panel on the right shows the selected node's parameters — edit them directly. Every parameter change instantly recomputes all downstream tensor shapes.
Use the Auto Layout button to tidy the canvas. Use Export .py to download the generated Keras code at any time.
Phase 3 — Training
Press ▶ Train. The platform validates the graph first — it checks for a connected Input→Output path, no shape errors, and no disconnected nodes. If validation passes, training begins.
The live modal shows epoch progress with loss curves. You can close it and the training continues in the background. If you click ▶ Train again mid-run, the previous run is cancelled and a new one starts.
Phase 4 — Results
After training completes, you are taken to the Results tab. The left panel shows metric cards, epoch progress bars, and filled loss/accuracy curves. The right panel shows the confusion matrix, per-class precision/recall/F1 table, and precision vs recall bar chart.
Run another experiment (change architecture → retrain) and the Results tab updates completely for the new run.
Phase 5 — Export
Click Export .py on the canvas toolbar to download the full Keras model code. Click ⬇ Export CSV or ⬇ Pipeline .py in the Dataset tab header to export the preprocessed data or the sklearn preprocessing pipeline.
Build Tab
The visual neural architecture editor
Palette
The left sidebar lists all available blocks, grouped by category. The three buttons at the top (📊 Tab / 🖼 Img / 〰 Seq) filter the palette to show only blocks valid for the current modality. Greyed blocks exist but are incompatible — they are shown so you can see what is available in other modalities.
Canvas
Drag any block from the palette onto the canvas. Nodes can be freely repositioned by dragging their header. Right port (→) is the output; left port (←) is the input. Draw connections by dragging from a right port to a left port.
Every edge displays the tensor shape flowing through it. A green shape label means valid. A red border on a node means a shape error — the error message is in the Inspector panel.
Delete a node by clicking the × in its header. This also removes all connected edges.
Inspector Panel
Click any node to select it and open its parameters in the right Inspector panel. All parameters are editable inline. Changes propagate immediately through the entire graph — downstream shapes update in real time. The Inspector also shows which nodes connect in and out of the selected node.
Code Mirror
The bottom of the Inspector panel shows the generated Python/Keras code for the current graph, updated live as you build. Click Export .py in the canvas toolbar to download it as a file.
⚠
Always include exactly one Input and one Output node. The validator will reject graphs that are missing either. The Input node must be the entry point with no incoming connections. The Output node must be the terminal with no outgoing connections.
💡
Templates are the fastest starting point. Click MLP, CNN, LSTM, or Transformer in the header to load a pre-wired graph. Then modify it — swap an activation, add a Dropout, change the Dense units — rather than building from scratch.
Dataset Tab
Ingestion, preprocessing, EDA, and binding
Modality Selection
The three tabs at the top of the Dataset panel (📊 Tabular / 🖼 Image / 〰 Sequence) determine the ingestion path. Select the one that matches your data before loading. Each modality shows a different left sidebar with modality-appropriate controls.
Loading Data
Tabular: drag-drop or browse CSV/JSON/TSV. Built-ins: Iris, Wine, Diabetes, Titanic. The last column is auto-selected as the target — change it in the Target Column selector.
Image: upload a ZIP file where each folder is a class name (e.g. cats/, dogs/). Built-ins: MNIST demo (10 classes) and Shapes (3 classes).
Sequence: upload a time-series CSV where rows are timesteps and columns are features. Built-ins: Sine+Noise, Synthetic Stock (OHLCV), ECG synthetic.
EDA Tabs
Six EDA tabs are available after loading:
• Overview — summary statistics, sample rows, processed shape
• Columns — per-column type/missing/outlier management with drop checkboxes
• Distributions — histogram + IQR boxplot per numeric feature
• Correlation — Pearson heatmap for first 8 numeric features
• Class Balance — imbalance ratio with severity verdict
• Validation — 7 automated checks including shape match with the current graph
Bind to Graph
The Bind to Graph → button at the bottom of the Dataset panel reads the processed feature count and class count and pushes them into the Input and Output nodes of the current graph. It also switches the Build tab to the matching modality. Always bind after loading your final dataset and before training.
⚠
The Validation tab tells you if there is a shape mismatch between your dataset and your graph's Input node. Fix it either by rebinding or manually editing the Input node's shape parameter.
Results Tab
Metrics, curves, confusion matrix, and per-class analysis
Left Panel — Metrics & Curves
Metric cards: Best Val Accuracy, Best Val Loss, Final Train Accuracy, Final Val Accuracy. Best Val Accuracy is highlighted green as the primary metric.
Training progress: per-epoch accuracy bars at sampled epoch intervals.
Loss curve: filled area chart with grid lines for both Train and Val loss. The gap between the two lines is a visual indicator of overfitting — a widening gap late in training means the model is memorising training data.
Accuracy curve: same treatment for Train and Val accuracy.
Right Panel — Confusion Matrix
The N×N confusion matrix shows predicted vs actual class for all classes. Green diagonal cells = correct predictions. Red off-diagonal cells = errors, intensity proportional to count. Class support bars alongside show how many samples exist per class.
Per-Class Metrics Table
Shows Precision, Recall, and F1 Score for every class as inline bar charts, plus raw TP/FP/FN counts. The macro average row at the bottom gives an unweighted mean across all classes.
Precision vs Recall Chart
Side-by-side bars per class comparing precision (solid) and recall (semi-transparent). A large gap between them signals a class where the model is either over-predicting or under-predicting.
Status Bar
The bar at the bottom persists best Val Acc and Val Loss across the session. Run multiple experiments — each overwrites the Results panel and updates the status bar.
💡
Reading the confusion matrix: a perfect classifier has non-zero values only on the diagonal. Look at which off-diagonal cells are largest — they tell you which classes the model most often confuses with each other.

Modalities
How data type determines what blocks are valid
📊 Tabular
Input shape: [features] — a flat 1D vector per sample. Example: Iris has 4 features → shape=4. Use Dense layers only. Typical architecture: Input → Dense → BatchNorm → Dropout → Dense → Output.
🖼 Image
Input shape: [H, W, C] — height × width × channels. Example: MNIST → 28,28,1. RGB → C=3. Use Conv2D → Pool → Conv2D → GlobalAvgPool2D → Dense → Output. Never connect Dense directly after Conv2D without Flatten or GlobalAvgPool2D first — shape inference will catch this.
〰 Sequence
Input shape: [timesteps, features] — a 2D tensor per sample. Example: 50-step window with 3 sensors → 50,3. Use Embedding (for text), Conv1D, or RNN blocks. To connect to Dense after an RNN: set return_sequences=False on the last RNN layer. To stack RNNs: set return_sequences=True on all but the last.
⚠
Switching modality changes what blocks are valid but does not delete nodes already on the canvas. If you switch from Image to Tabular with Conv2D nodes still present, those nodes will show shape errors. Clear the canvas first or use a template.
Tips & Gotchas
Common mistakes and how to avoid them
Shape Errors
⚠
Conv2D needs 3D input [H,W,C]. If you connect a Dense layer directly into a Conv2D, the edge turns red. Insert a Reshape block first, or reconsider the architecture.
⚠
LSTM needs 2D input [T,F]. If you try to feed a flat 1D vector into LSTM, it will error. Use Reshape to add a timestep dimension, or load a proper sequence dataset.
⚠
Add requires identical shapes on both inputs. The two branches you connect to an Add block must have the same tensor shape, or a shape error appears. Use Concat instead if shapes differ.
Architecture Advice
💡
GlobalAvgPool2D is better than Flatten before Dense in CNNs. It reduces parameters dramatically and regularises the network. Try both and compare Results.
💡
Add BatchNorm before activation, not after. The correct order is: Dense/Conv → BatchNorm → Activation → Dropout.
💡
For LSTM stacks, only the last LSTM should have return_sequences=False. All preceding LSTMs need return_sequences=True to pass the full sequence to the next layer.
💡
TransformerEncoder preserves shape [T,F]. After a stack of Transformer blocks, add GlobalAvgPool1D to collapse the sequence to [F] before connecting to Dense → Output.
Training
🟢
Start with Adam, lr=0.001. Only switch to SGD if you want to experiment with learning rate schedules. RMSProp is a good alternative for RNNs.
💡
If val loss rises while train loss falls, you are overfitting. Add Dropout (rate 0.3–0.5), reduce network width, or get more data.
💡
If both losses are high and flat, the learning rate may be too low, or the architecture is too shallow. Try increasing lr to 0.01 or adding Dense layers.
FAQ
Frequently asked questions
Why is the Train button disabled / showing an error? ▼
The validator checks four things before allowing training: (1) at least one Input node exists, (2) at least one Output node exists, (3) no node has a red shape error, (4) no node is disconnected (all nodes must be reachable from Input). Fix whichever condition is failing — the error toast tells you which one.
How do I connect two nodes? ▼
Click and drag from the small circle on the RIGHT side of a node (the output port) and release on the small circle on the LEFT side of another node (the input port). A blue curved edge will appear. If the connection is invalid due to shape mismatch, the edge turns red.
The Results tab shows old results even after a new run. ▼
This was a bug in earlier versions, fixed in v5. Each new training run clears the Results panel immediately when you click ▶ Train and shows "Training in progress…" until the new results are ready. If you see stale results, try clicking ▶ Train again — a concurrent run may have been in flight.
What does "IR = 5.2" mean in the status bar? ▼
IR is the Imbalance Ratio — the count of the largest class divided by the count of the smallest class. IR=1 means perfectly balanced. IR=5.2 means the majority class has 5.2× more samples than the minority class. Above 2 is moderate imbalance; above 5 is severe. Use the Class Imbalance control in the Dataset tab to oversample or use class weights.
Can I train on my own image dataset? ▼
Image ZIP ingestion currently requires a server-side backend. In the browser-only version, use the MNIST demo or Shapes built-in to test CNN architectures, then export the generated Keras code and run it locally with your own dataset using tf.keras.utils.image_dataset_from_directory.
How do I export my model or preprocessing code? ▼
Three exports are available: (1) Export .py from the canvas toolbar — downloads the Keras model architecture and compile/fit code. (2) Export CSV from the Dataset tab header — downloads the preprocessed dataset. (3) Pipeline .py from the Dataset tab header — downloads a complete sklearn preprocessing pipeline script matching your current settings.
The confusion matrix shows class names like "Class 0". How do I get real names? ▼
Load a dataset in the Dataset tab and click Bind to Graph before training. The platform reads the actual class labels from your dataset and uses them in the confusion matrix. If no dataset is bound, it falls back to generic "Class N" labels.
Training is simulated — how do I run real training? ▼
The current version simulates training curves for educational purposes. Export the .py file and run it locally with Python + TensorFlow installed. The generated code is valid Keras and will train on real data. Backend execution (real TF.js or Python server) is on the development roadmap.
Architecture Tutorials
Visual walkthroughs of MLP, CNN, LSTM, and Transformer
Multi-Layer Perceptron (MLP)
Best for: tabular classification & regression · modality: 📊 Tabular
Key Idea
Every neuron connects to every neuron
Each Dense layer is a learned linear transformation followed by a nonlinear activation. The network learns which feature combinations predict the target.
Why BatchNorm
Stabilises training
Normalises activations within each mini-batch. Prevents internal covariate shift — gradients flow more reliably. Place between Dense and activation.
Why Dropout
Prevents memorisation
Randomly zeros activations during training (rate=0.3 means 30% zeroed). Forces the network to learn redundant representations. Disabled at inference.
💡
In NeuralArc: load Iris or Diabetes, click Bind to Graph, then load the MLP template. The Input shape auto-adjusts to match your feature count.
Convolutional Neural Network (CNN)
Best for: image classification · modality: 🖼 Image
Key Idea
Local feature detection
Conv filters detect local patterns (edges, textures) regardless of position. Early layers detect simple features; deeper layers detect complex shapes.
Parameter sharing
Massively efficient
A 3×3 filter applied to a 28×28 image uses only 9 weights — the same kernel is reused at every spatial position. Far fewer params than a fully-connected layer.
Pooling
Spatial downsampling
MaxPool 2×2 halves H and W, reducing computation and building spatial invariance. GlobalAvgPool at the end is better than Flatten for reducing overfitting.
💡
In NeuralArc: switch palette to 🖼 Image, load the CNN template. Input is 28×28×1. Try changing filters from 32→64 in the first Conv2D and observe the shape propagation.
Long Short-Term Memory (LSTM)
Best for: time-series, sequences · modality: 〰 Sequence
Key Idea
Memory across time
LSTM maintains a cell state that carries information across many timesteps. Gates learn what to remember, what to forget, and what to output at each step.
4 Gates
Forget · Input · Cell · Output
Forget gate decides what to discard from memory. Input gate decides what new info to store. Cell candidate proposes updates. Output gate filters the hidden state.
Stacking LSTMs
return_sequences is critical
All LSTMs except the last must have return_sequences=True so they output the full [T, units] tensor to the next layer. The final LSTM uses False to collapse to [units].
💡
In NeuralArc: switch to 〰 Sequence, load the LSTM template. Notice the first LSTM has return_sequences=True and the second has False. Try switching and observe the shape error.
Transformer Encoder
Best for: long sequences, text, time-series · modality: 〰 Sequence
Key Idea
Attention is all you need
Instead of processing tokens one-by-one like RNNs, attention computes relationships between every pair of positions simultaneously. Parallelisable and handles long-range dependencies better.
Residual + LayerNorm
Enables deep stacking
Each sub-block adds its output back to its input (residual) and normalises. This allows stacking 6, 12, or even 96 encoder blocks without gradient vanishing.
Shape preservation
[T, d_model] in → out
TransformerEncoder preserves the sequence shape exactly. Add GlobalAvgPool1D after the last encoder block to collapse [T, d_model] → [d_model] before a Dense classifier.
💡
In NeuralArc: load the Transformer template. Stack 2–3 TransformerEncoder blocks, add GlobalAvgPool1D, then Dense → Output. Notice shapes stay [T, 64] through every encoder block.

About NeuralArc
NeuralArc Studio was conceived and built as a pedagogical tool for students learning neural networks — bridging the gap between abstract theory and practical implementation. Unlike black-box tools, NeuralArc exposes the full architecture graph with live tensor shape propagation, so students understand exactly what happens to their data at every layer.
The platform supports three data modalities (Tabular, Image, Sequence), 30+ layer types from basic Dense to full Transformer encoders, a complete data preprocessing pipeline, EDA with automated validation, and exports valid Keras code that students can take directly into a Python environment.
